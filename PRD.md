# **PRD — App Financeiro Pessoal**

## **1\. Visão geral**

Criar um sistema/app financeiro pessoal simples e intuitivo para registrar **entradas, saídas, contas a vencer, receitas fixas, relatórios com gráficos e projeção de saldo futuro**.

O objetivo é ajudar o usuário a saber:

* Quanto entrou no mês.  
* Quanto saiu no mês.  
* Quanto ainda vai pagar.  
* Qual será o saldo estimado nos próximos dias/meses.  
* Quais contas estão próximas do vencimento.

---

# **2\. Público-alvo**

Usuários que querem controlar finanças pessoais de forma simples, sem precisar usar planilhas complexas.

---

# **3\. Funcionalidades principais**

## **3.1 Cadastro/Login**

O sistema deve ter autenticação com Supabase.

Funcionalidades:

* Criar conta.  
* Fazer login.  
* Sair da conta.  
* Cada usuário vê apenas seus próprios dados.

---

## **3.2 Dashboard inicial**

Tela principal com resumo financeiro:

* Saldo atual.  
* Total de entradas no mês.  
* Total de saídas no mês.  
* Contas vencendo nos próximos dias.  
* Últimas movimentações.  
* Gráfico simples de receitas x despesas.

---

## **3.3 Registro de movimentações**

O usuário poderá cadastrar:

* Tipo: entrada ou saída.  
* Descrição.  
* Valor.  
* Categoria.  
* Data.  
* Observação opcional.

Exemplos:

Entrada:

* Salário  
* Pix recebido  
* Venda

Saída:

* Mercado  
* Energia  
* Internet  
* Cartão  
* Transporte

---

## **3.4 Receitas fixas**

Cadastro de receitas recorrentes, como:

* Salário  
* Bolsa  
* Aluguel recebido  
* Pagamento mensal

Campos:

* Nome da receita.  
* Valor.  
* Dia de recebimento.  
* Categoria.  
* Ativo/inativo.

---

## **3.5 Contas a vencer**

Cadastro de contas futuras:

* Nome da conta.  
* Valor.  
* Data de vencimento.  
* Categoria.  
* Status: pendente, paga ou atrasada.  
* Notificação visual no sistema.

Exemplo:

“Conta de energia vence em 3 dias.”

---

## **3.6 Relatórios com gráficos**

Relatórios por:

* Mês.  
* Categoria.  
* Tipo: entrada ou saída.  
* Comparativo receitas x despesas.

Gráficos:

* Gráfico de pizza por categoria.  
* Gráfico de barras por mês.  
* Gráfico de linha com evolução do saldo.

Pode usar biblioteca como **Chart.js**.

---

## **3.7 Projeção de saldo**

O sistema deve calcular uma previsão com base em:

* Saldo atual.  
* Receitas futuras cadastradas.  
* Contas pendentes.  
* Movimentações futuras.

Exemplo:

Saldo atual: R$ 1.000  
 Receitas futuras: R$ 2.000  
 Contas a pagar: R$ 1.200  
 Saldo projetado: R$ 1.800

---

## **3.8 Notificações**

Notificações dentro do sistema:

* Conta vence hoje.  
* Conta vence nos próximos 3 dias.  
* Conta está atrasada.  
* Receita prevista para os próximos dias.

No começo, pode ser apenas um alerta visual no dashboard.

---

# **4\. Estrutura inicial do banco Supabase**

## **Tabela: profiles**

create table profiles (  
 id uuid primary key references auth.users(id) on delete cascade,  
 nome text,  
 created\_at timestamp with time zone default now()  
);  
---

## **Tabela: categorias**

create table categorias (  
 id bigint generated by default as identity primary key,  
 user\_id uuid references auth.users(id) on delete cascade,  
 nome text not null,  
 tipo text check (tipo in ('entrada', 'saida')),  
 created\_at timestamp with time zone default now()  
);  
---

## **Tabela: movimentacoes**

create table movimentacoes (  
 id bigint generated by default as identity primary key,  
 user\_id uuid references auth.users(id) on delete cascade,  
 tipo text check (tipo in ('entrada', 'saida')) not null,  
 descricao text not null,  
 valor numeric(10,2) not null,  
 categoria text,  
 data date not null,  
 observacao text,  
 created\_at timestamp with time zone default now()  
);  
---

## **Tabela: receitas\_fixas**

create table receitas\_fixas (  
 id bigint generated by default as identity primary key,  
 user\_id uuid references auth.users(id) on delete cascade,  
 nome text not null,  
 valor numeric(10,2) not null,  
 dia\_recebimento int not null,  
 categoria text,  
 ativo boolean default true,  
 created\_at timestamp with time zone default now()  
);  
---

## **Tabela: contas**

create table contas (  
 id bigint generated by default as identity primary key,  
 user\_id uuid references auth.users(id) on delete cascade,  
 nome text not null,  
 valor numeric(10,2) not null,  
 vencimento date not null,  
 categoria text,  
 status text default 'pendente' check (status in ('pendente', 'paga', 'atrasada')),  
 created\_at timestamp with time zone default now()  
);  
---
